# Роадмап деплоя (Android APK, автономная работа)

Отдельный документ от [ROADMAP.md](./ROADMAP.md) (функционал продукта). Здесь — только сборка, раздача APK и автономная работа с Supabase.

---

## Зачем это нужно

| Режим | Что нужно игрокам |
|--------|-------------------|
| **Expo Go + твой `expo start`** | Телефон должен достучаться до твоего ПК (LAN/tunnel), одна версия Expo Go |
| **Установка по APK** | Только интернет; **Metro/ПК не нужны**. JS-бандл и нативные модули уже внутри APK |

Бэкенд уже **облачный** ([`src/lib/supabase.js`](./src/lib/supabase.js)) — «автономность» для игры = **отдельная установка приложения + доступ в интернет к Supabase**. Отдельный сервер игры не требуется.

**Доделывать весь функционал до первого APK не обязательно**, но имеет смысл пройти смоук-тест (логин, комната, игра, чат), чтобы не пересобирать APK из-за критичных багов.

---

## Что такое APK и как его получить (Expo SDK 54)

Рекомендуемый путь — **EAS Build** (облачная сборка Expo):

1. Аккаунт на [expo.dev](https://expo.dev) (бесплатного тарифа достаточно для пробных сборок с ограничениями).
2. В каталоге проекта: установить CLI `npm install -g eas-cli`, войти `eas login`.
3. Инициализация: `eas build:configure` — создаётся `eas.json` с профилями `development` / `preview` / `production`.
4. В `app.json` (или `app.config.js`) для Android задать уникальный **`android.package`** (например `com.yourname.gamemessenger`) — иначе сборка не пройдёт. При необходимости — `version` / `android.versionCode` для обновлений.
5. Сборка APK для раздачи «вручную»: профиль с `"android": { "buildType": "apk" }` (часто в `preview`), команда:
   - `eas build -p android --profile preview`
6. По завершении — ссылка на артефакт в дашборде EAS; скачиваешь **.apk**, передаёшь другу (мессенджер, облако).
7. На телефоне друга: **Настройки → безопасность → установка из неизвестных источников** (формулировки зависят от версии Android), открыть APK, установить.

**Play Store** обычно требует **AAB**, не APK — это отдельный профиль `production` и аккаунт разработчика Google Play (платный регистрационный взнос). Для «потестить с друзьями» достаточно APK.

Альтернатива: локальная сборка (`eas build --local`) — нужны Android SDK/NDK на машине; для первого раза проще облако.

## Автономная работа приложения после установки

- Приложение при старте обращается к **Supabase** по URL и anon key из кода (или из переменных окружения, если вынесешь в **EAS Secrets** и `app.config.js`).
- **Realtime, комнаты, чат** работают так же, как в Expo Go, без твоего компьютера.
- Обновления логики: новый билд (новый APK/AAB) и переустановка/обновление у пользователей.

Опционально позже: вынести ключи в `EXPO_PUBLIC_*` и не коммитить секреты в git.

## Чеклист деплоя

- [ ] Убедиться, что проект в порядке: `npx expo doctor`, при необходимости `npx expo install --fix`.
- [ ] Зарегистрироваться на expo.dev, `eas login`, выполнить `eas build:configure`.
- [ ] Добавить `android.package` и при необходимости имя/иконки в `app.json`.
- [ ] Настроить Supabase БД/Realtime/Storage: новая БД — `supabase_setup_v2.sql`; существующая БД — `scripts/migrate_existing_db_to_v2.sql` (запуск в Supabase → SQL Editor).
- [ ] В `eas.json` настроить профиль с **APK** для тестовых установок (`buildType: "apk"`).
- [ ] Запустить `eas build -p android --profile preview` (или выбранный профиль), дождаться артефакта, скачать APK.
- [ ] Раздать APK; на устройствах разрешить установку из неизвестных источников (для теста вне магазина).
- [ ] (По желанию) Вынести Supabase URL/key в `EXPO_PUBLIC_*` + EAS Secrets для смены без правки кода.

## Полезные команды (справочно)

```bash
cd GameMessengerMobile
npx expo doctor
eas login
eas build:configure
eas build -p android --profile preview
```

После первой успешной сборки артефакт смотри на [expo.dev](https://expo.dev) → проект → Builds.
